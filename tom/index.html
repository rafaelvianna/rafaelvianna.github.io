Summary

The aim of this project was to enable Trainline to expand their paid search activity into new markets quickly and without requiring significant extra resource.
With the marketing team I designed a solution which would combine Trainline’s internal data with data provided by Google and upload the results to DoubleClick where we could use the ‘Inventory Managment’ feature to auto-generate campaigns

Implementation

Internal data was stored on S3; external data on Google Cloud. I used Boto3 and Google Drive SDK to fetch the data; pandas / nltk / google translate to perform the transformations and the Google Drive sdk to load.

Cliteq Tech
Summary

As a fast growing agency Clicteq needed a way to create campaigns fast and accurately. They designed a campaign builder which allowed a user create an entire account,complete with millions of keywords and thousands of ads in seconds.

Implementation

I used Flask as the web framework, postgres as the database framework and AWS Elastic Beanstalk for the hosting. The actual process was written using pandas, pytest, numpy among others.
The most challenging aspect was reflecting the complex logic required where different levels of the hierarchy needed to adapt depending on the configuration set.
